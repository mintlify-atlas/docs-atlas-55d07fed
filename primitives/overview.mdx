Primitives are low-level, general-purpose UI components. Unlike [connected components](/connected-components/authenticator) — which wire directly into AWS services and handle auth or storage flows — primitives have no cloud dependencies. They are building blocks you compose to create any interface.

Every primitive is:

- **Accessible by default** — follows WAI-ARIA patterns with keyboard navigation and screen-reader support.
- **Themeable** — styled with [design tokens](/theming/design-tokens) that you can override globally or per-component.
- **Polymorphic** — accepts an `as` prop to change the underlying HTML element while keeping all style and event props.
- **Style-prop-aware** — accepts layout and spacing props (`padding`, `margin`, `width`, `gap`, etc.) directly, without writing CSS.

## Import pattern

All primitives are exported from `@aws-amplify/ui-react`. Import only what you need.

```tsx
import { Button, TextField, Flex, Badge } from '@aws-amplify/ui-react';
```

## Common props

Every primitive extends the base props below in addition to its own specific props.

| Prop        | Type                  | Description                                     |
| ----------- | --------------------- | ----------------------------------------------- |
| `as`        | `React.ElementType`   | Change the rendered HTML element.               |
| `className` | `string`              | Append a CSS class to the component.            |
| `style`     | `React.CSSProperties` | Inline styles via a React CSSProperties object. |
| `id`        | `string`              | HTML `id` attribute.                            |
| `testId`    | `string`              | Sets `data-testid` for testing.                 |
| `isDisabled`| `boolean`             | Disables the element.                           |

Style shorthand props like `padding`, `margin`, `width`, `gap`, `color`, `backgroundColor`, and more are also available on every primitive and map directly to their CSS equivalents.
