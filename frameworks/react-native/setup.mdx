This guide walks through installing and configuring `@aws-amplify/ui-react-native` in a React Native project. The package requires `react-native-safe-area-context` as a peer dependency, which must be linked for both iOS and Android.

## Installation

<Steps>
  <Step title="Install the Amplify UI package and its peer dependencies">
    <Tabs>
      <Tab title="npm">
        ```bash
        npm install @aws-amplify/ui-react-native aws-amplify react-native-safe-area-context
        ```
      </Tab>
      <Tab title="yarn">
        ```bash
        yarn add @aws-amplify/ui-react-native aws-amplify react-native-safe-area-context
        ```
      </Tab>
    </Tabs>

    <Warning>
      `react-native-safe-area-context` is a required peer dependency. The `Authenticator` component uses safe area insets to position content correctly on devices with notches or home indicators. The app will crash at runtime if this package is not installed and linked.
    </Warning>
  </Step>

  <Step title="Link native modules (iOS)">
    If you are using a bare React Native project (not Expo), install the CocoaPods for iOS:

    ```bash
    npx pod-install
    ```

    Expo managed workflow links native modules automatically — no additional step is required.
  </Step>

  <Step title="Configure Amplify">
    Call `Amplify.configure()` at the top level of your app before any component renders. The `amplify_outputs.json` file is generated by the Amplify CLI or Amplify Studio.

    ```tsx index.js
    import { AppRegistry } from 'react-native';
    import { Amplify } from 'aws-amplify';
    import outputs from './amplify_outputs.json';
    import App from './App';
    import { name as appName } from './app.json';

    Amplify.configure(outputs);

    AppRegistry.registerComponent(appName, () => App);
    ```
  </Step>

  <Step title="Wrap your app with SafeAreaProvider">
    `react-native-safe-area-context` requires a `SafeAreaProvider` at the root of your component tree:

    ```tsx App.tsx
    import { SafeAreaProvider } from 'react-native-safe-area-context';
    import { Authenticator } from '@aws-amplify/ui-react-native';

    export default function App() {
      return (
        <SafeAreaProvider>
          <Authenticator.Provider>
            <Authenticator>
              {({ signOut, user }) => (
                <MainScreen user={user} signOut={signOut} />
              )}
            </Authenticator>
          </Authenticator.Provider>
        </SafeAreaProvider>
      );
    }
    ```
  </Step>

  <Step title="Run your app">
    <Tabs>
      <Tab title="iOS">
        ```bash
        npx react-native run-ios
        ```
      </Tab>
      <Tab title="Android">
        ```bash
        npx react-native run-android
        ```
      </Tab>
      <Tab title="Expo">
        ```bash
        npx expo start
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Authenticator usage

The `Authenticator` component renders the full sign-in and sign-up flow. Wrap it in `Authenticator.Provider` to enable the `useAuthenticator` hook in child components.

```tsx App.tsx
import { Authenticator, useAuthenticator } from '@aws-amplify/ui-react-native';
import { SafeAreaProvider } from 'react-native-safe-area-context';
import { View, Text, Button, StyleSheet } from 'react-native';

function HomeScreen() {
  const { user, signOut } = useAuthenticator();

  return (
    <View style={styles.container}>
      <Text>Hello, {user?.username}</Text>
      <Button title="Sign out" onPress={signOut} />
    </View>
  );
}

export default function App() {
  return (
    <SafeAreaProvider>
      <Authenticator.Provider>
        <Authenticator>
          <HomeScreen />
        </Authenticator>
      </Authenticator.Provider>
    </SafeAreaProvider>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

## withAuthenticator HOC

For simpler setups, use the `withAuthenticator` higher-order component to gate your root component behind authentication:

```tsx App.tsx
import { withAuthenticator } from '@aws-amplify/ui-react-native';
import { View, Text, StyleSheet } from 'react-native';

function App() {
  return (
    <View style={styles.container}>
      <Text>You are signed in.</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center' },
});

export default withAuthenticator(App);
```

## Applying a theme

Use `ThemeProvider` and `createTheme` to apply a custom theme. Unlike web, all values must be valid React Native style values:

```tsx App.tsx
import { ThemeProvider, createTheme, Authenticator } from '@aws-amplify/ui-react-native';
import { SafeAreaProvider } from 'react-native-safe-area-context';

const theme = createTheme({
  tokens: {
    colors: {
      brand: {
        primary: { 80: { value: '#007bff' } },
      },
    },
  },
});

export default function App() {
  return (
    <SafeAreaProvider>
      <ThemeProvider theme={theme}>
        <Authenticator.Provider>
          <Authenticator>{/* your app */}</Authenticator>
        </Authenticator.Provider>
      </ThemeProvider>
    </SafeAreaProvider>
  );
}
```

## Expo setup

Expo managed workflow handles native linking automatically. Install the packages and configure Amplify:

```bash
npx expo install @aws-amplify/ui-react-native aws-amplify react-native-safe-area-context
```

<Note>
  Use `npx expo install` instead of `npm install` to ensure Expo-compatible versions of native packages are selected.
</Note>
