This guide walks you through adding a fully functional sign-in and sign-up flow to a React app using the `Authenticator` component. The `Authenticator` handles the entire authentication UI тАФ sign in, sign up, forgot password, MFA тАФ with zero custom code.

<Note>
  This quickstart uses React. For Angular, Vue, or React Native examples, see the [Frameworks](/frameworks/react/overview) section.
</Note>

## Prerequisites

- Node.js 18 or later
- An [AWS Amplify](https://docs.amplify.aws/) backend with Auth configured, which produces an `amplify_outputs.json` file

<Note>
  The `amplify_outputs.json` file is generated by the Amplify CLI or Amplify console and contains your backend configuration. If you don't have a backend yet, follow the [Amplify getting started guide](https://docs.amplify.aws/react/start/) to create one.
</Note>

## Build your first authenticated app

<Steps>

<Step title="Install dependencies">

Create a new React app (or use an existing one) and install Amplify UI for React along with `aws-amplify`:

```bash
npm create vite@latest my-app -- --template react-ts
cd my-app
npm install @aws-amplify/ui-react aws-amplify
```

</Step>

<Step title="Copy your backend configuration">

Copy the `amplify_outputs.json` file from your Amplify backend into the root of your project:

```
my-app/
тФЬтФАтФА amplify_outputs.json   тЖР copy here
тФЬтФАтФА src/
тФВ   тФЬтФАтФА main.tsx
тФВ   тФФтФАтФА App.tsx
тФФтФАтФА package.json
```

<Note>
  `amplify_outputs.json` contains non-secret configuration values (API endpoints, region, user pool IDs). It is safe to commit to source control.
</Note>

</Step>

<Step title="Configure Amplify">

In your app entry point, import and call `Amplify.configure()` with your outputs file. This must run before any Amplify API calls are made.

```tsx src/main.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { Amplify } from 'aws-amplify';
import '@aws-amplify/ui-react/styles.css';
import outputs from '../amplify_outputs.json';
import App from './App';

Amplify.configure(outputs);

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

</Step>

<Step title="Add the Authenticator">

Wrap your app content with the `Authenticator` component. It renders the sign-in and sign-up UI automatically, and only shows your app content once the user is authenticated.

Choose between using `Authenticator` as a wrapper component or the `withAuthenticator` higher-order component (HOC):

<Tabs>
  <Tab title="JSX (component)">
    ```jsx src/App.jsx
    import { Authenticator } from '@aws-amplify/ui-react';

    export default function App() {
      return (
        <Authenticator>
          {({ signOut, user }) => (
            <main>
              <h1>Hello, {user?.username}</h1>
              <button onClick={signOut}>Sign out</button>
            </main>
          )}
        </Authenticator>
      );
    }
    ```
  </Tab>
  <Tab title="TSX (component)">
    ```tsx src/App.tsx
    import { Authenticator } from '@aws-amplify/ui-react';

    export default function App() {
      return (
        <Authenticator>
          {({ signOut, user }) => (
            <main>
              <h1>Hello, {user?.username}</h1>
              <button onClick={signOut}>Sign out</button>
            </main>
          )}
        </Authenticator>
      );
    }
    ```
  </Tab>
  <Tab title="JSX (HOC)">
    ```jsx src/App.jsx
    import { withAuthenticator } from '@aws-amplify/ui-react';

    function App({ signOut, user }) {
      return (
        <main>
          <h1>Hello, {user?.username}</h1>
          <button onClick={signOut}>Sign out</button>
        </main>
      );
    }

    export default withAuthenticator(App);
    ```
  </Tab>
  <Tab title="TSX (HOC)">
    ```tsx src/App.tsx
    import type { AuthUser } from 'aws-amplify/auth';
    import type { UseAuthenticator } from '@aws-amplify/ui-react-core';
    import { withAuthenticator } from '@aws-amplify/ui-react';

    type AppProps = {
      signOut?: UseAuthenticator['signOut'];
      user?: AuthUser;
    };

    function App({ signOut, user }: AppProps) {
      return (
        <main>
          <h1>Hello, {user?.username}</h1>
          <button onClick={signOut}>Sign out</button>
        </main>
      );
    }

    export default withAuthenticator(App);
    ```
  </Tab>
</Tabs>

</Step>

<Step title="Run your app">

Start the development server:

```bash
npm run dev
```

Open `http://localhost:5173` in your browser. You will see the Amplify UI sign-in form. Create an account, verify your email, and sign in тАФ your app content renders once authentication is complete.

</Step>

</Steps>

## Complete minimal example

The following is a self-contained example of a fully authenticated React app with TypeScript:

```tsx src/main.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { Amplify } from 'aws-amplify';
import { Authenticator } from '@aws-amplify/ui-react';
import '@aws-amplify/ui-react/styles.css';
import outputs from '../amplify_outputs.json';

Amplify.configure(outputs);

function App() {
  return (
    <Authenticator>
      {({ signOut, user }) => (
        <main style={{ padding: '2rem' }}>
          <h1>Welcome, {user?.username}</h1>
          <button onClick={signOut}>Sign out</button>
        </main>
      )}
    </Authenticator>
  );
}

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

## What the Authenticator provides

Out of the box, `Authenticator` handles:

- Sign in with username, email, or phone number
- New user registration (sign up)
- Email or SMS verification
- Forgot password / reset password flow
- Multi-factor authentication (TOTP, SMS)
- Force new password (on first sign-in)

All flows are driven by your Amplify Auth backend configuration тАФ no additional setup is needed in the component.

<Tip>
  You can customise the form fields, override individual UI slots, add social providers (Google, Facebook, Apple), and apply your own theme. See the [Authenticator docs](/connected-components/authenticator) for the full API.
</Tip>
